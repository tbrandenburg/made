## Problem Statement

The Vite dev server runs over HTTP/1.1, which limits browsers to **6 concurrent connections per origin**. With 115 module requests on page load, 84 requests queue behind the limit for 200ms–1.1s each, wasting 60+ seconds total. HTTP/2 multiplexes all requests over a single connection, eliminating the bottleneck entirely.

**Current state (as of 2026-06-02):** The secondary fix (`optimizeDeps.include` for `marked`, `dompurify`, `react-virtuoso`) was already applied in commit `2eee871` (#449). The primary fix (HTTP/2 via `server.https: true` + `@vitejs/plugin-basic-ssl`) and remaining `optimizeDeps` entries (`@heroicons/react/24/outline`, `@xterm/xterm`, `@xterm/addon-fit`) are still missing.

## Analysis

### Change Rationale

HTTP/2 multiplexes all requests over a single TCP connection, eliminating the 6-connection bottleneck. Additionally, adding `@heroicons/react/24/outline`, `@xterm/xterm`, and `@xterm/addon-fit` to `optimizeDeps.include` pre-bundles these barrel-imported libraries to reduce individual file requests further.

### Evidence Chain

WHY: 84/115 requests are blocked before they can start, causing ~22s waterfall
↓ BECAUSE: Browsers cap concurrent HTTP/1.1 connections at 6 per origin
Evidence: `packages/frontend/vite.config.ts:1-35` — plain HTTP, no `server.https`, no HTTP/2

↓ BECAUSE: Vite serves unbundled source modules as individual HTTP/1.1 requests
Evidence: `packages/frontend/vite.config.ts:5` — only `@vitejs/plugin-react` in plugins array; no SSL plugin

↓ ROOT CAUSE: `server.https` and `@vitejs/plugin-basic-ssl` not configured; plus `@heroicons/react`, `@xterm/xterm`, `@xterm/addon-fit` not in `optimizeDeps.include`
Evidence: `packages/frontend/vite.config.ts:1-35` — current file has partial `optimizeDeps` (marked, dompurify, react-virtuoso) but missing the remaining heavy deps

## Implementation Plan

### Step 1: Add `@vitejs/plugin-basic-ssl` to devDependencies

**File**: `packages/frontend/package.json`
**Action**: UPDATE

**Current code (relevant section):**

```json
    "@vitejs/plugin-react": "^6.0.0",
```

**Required change:** Insert after `@vitejs/plugin-react`:

```json
    "@vitejs/plugin-basic-ssl": "^2.0.0",
    "@vitejs/plugin-react": "^6.0.0",
```

**Why**: HTTP/2 in browsers requires HTTPS; this plugin auto-generates a self-signed dev certificate with zero config.

---

### Step 2: Enable HTTPS + HTTP/2 and complete `optimizeDeps`

**File**: `packages/frontend/vite.config.ts`
**Lines**: 1-35
**Action**: UPDATE

**Current code:**

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
  // Pre-bundle heavy deps at startup to avoid lazy esbuild round-trips on first detail-page visit.
  // Keep this list updated when adding new heavy npm deps used by lazy-loaded routes.
  optimizeDeps: {
    include: ["marked", "dompurify", "react-virtuoso"],
  },
  server: {
    port: 5173,
    host: "0.0.0.0",
    // ✅ allow ngrok tunnels to access the dev server
    allowedHosts: [
      ".ngrok-free.dev", // current default ngrok domain
      ".ngrok.io", // legacy domain
      ".ngrok.app", // alternate domain
      ".org", // org domains
    ],
    proxy: {
      "/api": {
        target: "http://localhost:3000",
        changeOrigin: true,
        ws: true,
      },
    },
  },
});
```

**Required change:**

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import basicSsl from "@vitejs/plugin-basic-ssl";

export default defineConfig({
  plugins: [react(), basicSsl()],
  // Pre-bundle heavy deps at startup to avoid lazy esbuild round-trips on first detail-page visit.
  // Keep this list updated when adding new heavy npm deps used by lazy-loaded routes.
  optimizeDeps: {
    include: [
      "@heroicons/react/24/outline",
      "@xterm/xterm",
      "@xterm/addon-fit",
      "marked",
      "dompurify",
      "react-virtuoso",
    ],
  },
  server: {
    port: 5173,
    host: "0.0.0.0",
    https: true, // enables HTTP/2 — eliminates 6-connection HTTP/1.1 bottleneck
    // ✅ allow ngrok tunnels to access the dev server
    allowedHosts: [
      ".ngrok-free.dev", // current default ngrok domain
      ".ngrok.io", // legacy domain
      ".ngrok.app", // alternate domain
      ".org", // org domains
    ],
    proxy: {
      "/api": {
        target: "http://localhost:3000",
        changeOrigin: true,
        ws: true,
      },
    },
  },
});
```

**Why**: `basicSsl()` provides a self-signed cert → `server.https: true` enables HTTPS → modern browsers negotiate HTTP/2 automatically. Extended `optimizeDeps.include` pre-bundles remaining heavy barrel imports.

## Edge Cases & Risks

| Risk / Edge Case                              | Mitigation                                                                    |
| --------------------------------------------- | ----------------------------------------------------------------------------- |
| Browser shows self-signed cert warning        | One-time "Proceed anyway" in dev; document in AGENTS.md                       |

| ngrok tunnel with HTTPS cert negotiation      | `allowedHosts` already configured; no extra change needed                     |
| Production build unaffected                   | `server.https` only applies to `vite dev`, not `vite build`                   |
