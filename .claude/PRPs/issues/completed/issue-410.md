## Problem Statement

The `axios` transitive dependency (pulled in by `wait-on@9.0.4`) is pinned at `>=1.15.0`, but versions 1.0.0 - 1.15.1 have multiple HIGH severity CVEs (prototype pollution, NO_PROXY bypass, header injection, etc.). The override needs to be raised to `>=1.15.2` to force installation of a patched version.

---

## Analysis

### Change Rationale

The current override `">=1.15.0"` is too permissive — it allows npm to resolve to vulnerable versions (1.15.0 and 1.15.1). The fix is trivial: bump the override to `">=1.15.2"` so npm resolves to the first patched version.

### Evidence Chain

WHY: npm audit reports 1 HIGH severity vulnerability in axios
↓ BECAUSE: The `package.json` override allows versions 1.0.0 - 1.15.1, all of which are vulnerable
Evidence: `package.json:25` - `"axios": ">=1.15.0"`

↓ ROOT CAUSE: Override version constraint is too low
Evidence: `package.json:25` - `"axios": ">=1.15.0"` should be `"axios": ">=1.15.2"`

### Affected Files

| File            | Lines       | Action | Description                                    |
| --------------- | ----------- | ------ | ---------------------------------------------- |
| `package.json`  | 25          | UPDATE | Bump axios override from `>=1.15.0` to `>=1.15.2` |
| `package-lock.json` | (auto) | UPDATE | Regenerated by `npm install` after override change |

### Integration Points

- `wait-on@9.0.4` (devDependency in root `package.json:22`) is the direct consumer that pulls in axios
- No source code directly imports axios — it is purely transitive via `wait-on`
- No other `package.json` files in the monorepo reference axios

## Implementation Plan

### Step 1: Update axios override in root package.json

**File**: `package.json`
**Lines**: 25
**Action**: UPDATE

**Current:**

```json
    "axios": ">=1.15.0",
```

**Required change:**

```json
    "axios": ">=1.15.2",
```

**Why**: Versions 1.15.0 and 1.15.1 contain multiple HIGH severity vulnerabilities. 1.15.2 is the first patched version.

### Step 2: Reinstall dependencies to update lockfile

**Action**: Run `rm -rf node_modules package-lock.json && npm install` to regenerate `package-lock.json` with the resolved patched axios version.
